Setup Guide

Option A — WAHA Docker FREE works for any country

WAHA is a free open-source WhatsApp HTTP API. It runs on your computer in Docker and lets the test suite read OTPs automatically.

Step 1 — Install Docker Desktop

  • Go to docker.com/products/docker-desktop and download Docker Desktop for your operating system (Windows / Mac / Linux).
  • Install it and open the Docker Desktop app. Wait until you see Docker is running in the system tray / menu bar.
  • Verify in a terminal: docker --version (should print a version number)

Step 2 — Start WAHA

Copy and paste this single command into your terminal:
docker run -d --name waha -p 3000:3000 devlikeapro/waha

-d = runs in background  ·  --name waha = gives it a name  ·  -p 3000:3000 = opens port 3000  ·  devlikeapro/waha = the free WAHA image

Step 3 — Scan the QR code

  • Open http://localhost:3000/dashboard in your browser.
  • Click Start session if there is no active session.
  • A QR code will appear. Open WhatsApp on your test phone → Settings → Linked Devices → Link a Device → scan the QR code.
  • Once connected you will see Session WORKING. You only need to do this once — the session is saved automatically.

Step 4 — Configure your .env

PROD_OTP_BACKEND=waha
WAHA_URL=http://localhost:3000
WAHA_SESSION=default
WAHA_CHAT_ID=+88098976564@c.us   # replace with your test number
PROD_COUNTRY_CODE=+880
PROD_TEST_PHONE=98976564

The chat ID format is +{country_code}{local_number}@c.us. For Bangladesh: +88098976564@c.us

Management commands

Stop WAHA

docker stop waha

Start again

docker start waha

View logs

docker logs waha -f

Remove & restart fresh

docker rm -f waha && docker run ...

Option B — whatsapp-web.js Node.js bridge FREE

An open-source Node.js library that connects WhatsApp via QR scan and runs a local HTTP server the test suite can query for OTPs.

  • Install Node.js 18+ from nodejs.org if you don't have it. Verify: node --version
  • Open a terminal in this project's scripts/ folder: 
    cd scripts
npm install
  • Start the bridge: 
    node whatsapp_otp_server.js
    A QR code appears in the terminal. Scan it with WhatsApp (Settings → Linked Devices → Link a Device).
  • Once you see WhatsApp client ready, the bridge is running at http://localhost:3001/otp.
  • Set in .env: 
    PROD_OTP_BACKEND=whatsapp_local
WA_OTP_PORT=3001
🔒 The session is saved to scripts/.wwebjs_auth/. You won't need to scan the QR again unless you log out or delete that folder.

CI/CD (GitHub Actions)

How WAHA works in GitHub Actions

The CI workflow automatically starts a WAHA Docker container for the main test jobs. Docker is already installed on GitHub Actions Ubuntu runners — no extra setup needed.

How CI picks the OTP backend

  • Staging tests (default) — BASE_URL=dev.mehadedu.com → fixed OTP 123456 is used. WAHA starts but isn't needed.
  • Production tests — Set BASE_URL=https://mehadedu.com/en in the workflow dispatch. Also set PROD_OTP_BACKEND=waha and configure the secrets below.

GitHub Secrets to configure

Go to your repository → Settings → Secrets and variables → Actions → New repository secret.

Secret name Example value Description
PROD_OTP_BACKENDwahaWhich OTP system to use
WAHA_CHAT_ID+88098976564@c.usWhatsApp chat ID of the test phone
WAHA_SESSIONdefaultWAHA session name (leave as default)
PROD_TEST_PHONE98976564Local part of the production test number
PROD_COUNTRY_CODE+880Country code for production number

Important: WAHA session in CI

🔒 Each CI run starts a fresh WAHA container with no WhatsApp session. For production OTP in CI, you have two options:

Option 1 (recommended) — Self-host a permanent WAHA instance, set WAHA_URL secret to your server URL (e.g. https://waha.yourserver.com). The CI job will use your always-on session.

Option 2 — Keep testing against dev.mehadedu.com staging (no real OTP needed — always uses 123456).
